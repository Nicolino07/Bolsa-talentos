1) INICIAR TODOS LOS SERVICIOS
------------------------------
docker compose up -d

(-d oculta los logs)
(Levanta: backend, frontend, base de datos, prolog, pgadmin)

2) VER CONTENEDORES ACTIVOS
---------------------------
docker ps

3) VER LOGS DE UN SERVICIO
--------------------------
Backend:
    docker logs -f jobs_backend
Base de datos:
    docker logs -f jobs_db

4) DETENER TODO
---------------
docker compose down

5) REINICIAR TODO (SIN RECONSTRUIR)
-----------------------------------
docker compose down
docker compose up -d

6) RECONSTRUIR (CUANDO CAMBIAN Dockerfile, requirements o package.json)
------------------------------------------------------------------------
docker compose up --build -d

7) ENTRAR DENTRO DE UN CONTENEDOR
---------------------------------
Backend (FastAPI, Python):
    docker exec -it jobs_backend bash

Prolog:
    docker exec -it jobs_prolog bash
    luego dentro: swipl

Base de datos (psql):
    docker exec -it jobs_db bash
    psql -U postgres -d bolsa_talentos

8) RESETEAR COMPLETAMENTE (BORRA BASE DE DATOS)
-----------------------------------------------
docker compose down -v
docker compose up --build -d

9) ACTUALIZAR C√ìDIGO SIN RECONSTRUIR
------------------------------------
Si solo cambiaste .py, .js, .html, .css:
    Backend se autorecarga o:
        docker restart jobs_backend
    Front actualiza solo si se usa Vite
